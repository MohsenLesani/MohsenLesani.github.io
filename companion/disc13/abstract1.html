Constructing Bug Trace from Bug Specification

Mohsen Lesani, Jens Palsberg
 

We present an automatic tool that given a synchronization algorithm and a bug specification, constructs a trace that exhibits the bug. We define a denotational semantics for the set of histories that a program can produce. Based on the semantics, we represent both the program and the bug specification as constraints and use Z3 SMT solver to solve the constraints. If Z3 finds a model for the constraints, we realize it as an interleaving trace and represent it to the user in a graphical user interface.


We identify two problems that lead to non-opacity of transactional memory algorithms. Using our tool, we show that the well-known TM algorithms DSTM and McRT don’t satisfy opacity. DSTM suffers from a write-skew anomaly, while McRT suffers from a write-exposure anomaly.
